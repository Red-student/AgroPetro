Вопрос 1: Почему при создании новой БД появляется два админ-аккаунта?
---------------------------------------------------------------

Причина появления двух админов — дублирование логики добавления администратора:

1. В миграции (InitialCreate) и в OnModelCreating (EcologyDbContext) через HasData добавляется админ:
   - UserId = 1, UserName = "admin", Email = "admin@example.com", Password = "admin123", IsAdmin = true
2. В Program.cs при первом запуске, если базы нет, также вручную добавляется админ с теми же данными:
   - context.Users.Add(adminUser);

В результате, если база создаётся через миграции, EF Core применяет HasData и добавляет админа.
Если база создаётся через EnsureCreated() (Program.cs), админ добавляется вручную, а затем при миграции — второй раз.

**Рекомендация:**
Оставить только один способ добавления администратора (либо HasData, либо вручную в Program.cs), чтобы избежать дублей.


Вопрос 2: Происходит ли автоматическое обновление данных каждый час после запуска приложения?
---------------------------------------------------------------------

Нет, автоматическое обновление данных происходит раз в 24 часа (раз в сутки), а не каждый час.

- За это отвечает фоновая служба GrainDataUpdateService (Services/GrainDataUpdateService.cs):
  - Интервал обновления: private readonly TimeSpan _updateInterval = TimeSpan.FromHours(24);
  - После запуска приложения данные обновляются автоматически раз в сутки через XlsImporter.

**Если нужно обновлять каждый час — измените значение _updateInterval на TimeSpan.FromHours(1).**


---

Итого:
- Два админа появляются из-за дублирования логики добавления.
- Автоматическое обновление данных происходит раз в сутки, а не каждый час.
